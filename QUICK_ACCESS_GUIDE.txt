RECOMMENDATION #1: AUTO-BACKUP BEFORE RESTORE
--------------------------------------------------------------------------------
HOW TO ACCESS:
  1. Login as Administrator
  2. Click "Settings" in sidebar
  3. Click "Backup & Restore" tab
  4. Scroll to "Restore Database" section

HOW IT WORKS:
  1. Click "Choose File" and select backup .sql file
  2. System automatically creates safety backup first
  3. You'll see warning: "A safety backup will be created automatically"
  4. Click "Restore Database"
  5. Success message shows: Safety backup filename (e.g., smartiso_safety_20251216.sql)
  6. If restore fails, you can restore the safety backup

FILES: app/Controllers/Admin/Settings.php, app/Views/admin/settings/backup.php


RECOMMENDATION #2: ISO FORM REGISTRATION VALIDATION (Form Code, Revision No., Effectivity Date)
--------------------------------------------------------------------------------
HOW TO ACCESS (TAU-DCO User):
  1. Login with TAU-DCO account (user_type = 'tau_dco')
  2. Click "TAU-DCO" in sidebar
  3. Click "Form Approval" submenu
  4. View all forms with DCO status

APPROVAL PROCESS:
  1. Review form details: Form Code, Revision No., Effectivity Date
  2. Click "Approve" button to validate ISO registration
  3. Click "Revoke" to remove approval
  4. Status badge updates: Green (Approved) or Red (Pending)

REQUESTOR VIEW:
  1. Non-approved forms show "Pending DCO Approval" badge
  2. "Fill Out" button is disabled/locked
  3. Forms become available after DCO approval

TOGGLE FEATURE ON/OFF:
  1. Admin: Go to Settings > System Settings
  2. Find "Require DCO Approval" card
  3. Toggle switch ON/OFF

FILES: app/Controllers/Admin/DCOApproval.php, app/Views/admin/dco_approval/index.php


RECOMMENDATION #3: TEMPLATE REVISION MANAGEMENT (Active/Inactive Status)
--------------------------------------------------------------------------------
HOW TO ACCESS:
  1. Login as Administrator
  2. Click "Form Management" in sidebar
  3. Click "Form Templates"
  4. View Status column: Active (green) or Inactive (red)

HOW TO ACTIVATE/DEACTIVATE:
  1. Click "Actions" dropdown on template row
  2. Select "Activate" or "Deactivate"
  3. Confirm in modal
  4. Only ACTIVE templates appear to requestors
  5. Inactive templates are hidden but preserved

USE CASES:
  - Retire old form revisions: Deactivate old, activate new
  - Seasonal forms: Activate/deactivate as needed
  - Only one revision active at a time (best practice)

FILES: app/Models/Panel.php, app/Controllers/Admin/FormTemplates.php

RECOMMENDATION #5: AUDIT TRAIL AND ACTIVITY LOGGING
--------------------------------------------------------------------------------
HOW TO ACCESS:
  1. Login as Administrator
  2. Click "Audit Logs" in sidebar (under ADMINISTRATION)

HOW TO USE:

WHAT'S LOGGED:
  - Logins, logouts, password changes
  - Form submissions, approvals, rejections
  - User CRUD operations
  - Schedule changes
  - Database backup/restore
  - Configuration changes

RECOMMENDATION #6: GROUP FIELDS IN FORMS (Section Headers)
--------------------------------------------------------------------------------
HOW TO CREATE:
  1. Login as Administrator
  2. Go to "Form Management" > "Form Builder"
  3. Select form to edit or create new
  4. Click "Add Field" button
  5. Field Type dropdown: Select "Section Header"
  6. Label: Enter section title (e.g., "Personal Information")
  7. Default Value: Optional description text
  8. Field Width: 12 (full width recommended)
  9. Click "Add Field"
  10. Drag to position above related fields
  11. Save form

RESULT:
  - Section header appears as styled divider
  - Large bold title with icon
  - Optional description below
  - Groups related fields visually

FILES: app/Views/admin/dynamicforms/form_builder.php, app/Views/forms/view.php


RECOMMENDATION #7: APPROVED REQUESTS VIEWABLE AS REALISTIC ISO FORM

RECOMMENDATION #8: FEEDBACK REDESIGN (Star Ratings)
--------------------------------------------------------------------------------
HOW TO GIVE FEEDBACK:
  1. Login as requestor
  2. Go to "My Requests"
  3. Find COMPLETED request
  4. Click "Give Feedback" button
  5. Rate 5 categories by clicking stars (1-5):
     - Overall Experience
     - Service Quality
     - Timeliness
     - Staff Professionalism
     - Overall Satisfaction
  6. Add optional comments
  7. Click "Submit Feedback"

STAR INTERACTION:
  - Hover: Stars highlight in gold
  - Click: Stars fill up to selected rating
  - Change: Click different star to adjust
  - 5 stars = Excellent, 1 star = Very Poor

ADMIN VIEW:
  1. Go to "Analytics" or "Feedback"
  2. View aggregated ratings (e.g., 4.2/5.0 average)
  3. Filter by date, service, department
  4. Export to CSV

FILES: app/Views/feedback/create.php, app/Controllers/Feedback.php


RECOMMENDATION #9: PRINTABLE APPROVED/COMPLETED SERVICES
--------------------------------------------------------------------------------
HOW TO PRINT:
  1. Open any submission in document view
  2. Click "Print" button OR press Ctrl+P (Cmd+P on Mac)
  3. Print preview shows clean layout:
     ✓ Header, DCN, form content, signatures, QR code
     ✗ No sidebar, menus, or buttons
  4. Select printer or "Save as PDF"
  5. Adjust settings (color, pages, orientation)
  6. Click Print/Save

SAVE AS PDF:
  Windows: Ctrl+P → Destination: "Save as PDF" → Save
  Mac: Cmd+P → PDF dropdown → "Save as PDF"

WHAT PRINTS:
  ✓ Organization header, ISO control header, all fields, signatures, QR code
  ✗ Navigation, buttons, backgrounds (hidden via @media print CSS)

FILES: app/Views/forms/view_submission.php (print styles included)

  3. approver_signature_date: Approver signature date
  4. service_staff_signature_date: Staff signature date
  5. requestor_signature_date: Requestor signature date
  6. completion_date: When service completed
  7. updated_at: Last modification
  8. scheduled_date/time: Appointment schedule

WHERE TO VIEW:
  - Submission details page: All timestamps displayed
  - Signature blocks: Date below each signature
  - Status timeline: Visual timeline with dates
  - Reports: Processing time calculations

PROCESSING TIME:
  - Request to Approval: approved_at - created_at
  - Approval to Completion: completion_date - approved_at
  - Total Time: completion_date - created_at

FILES: Database table form_submissions (DATETIME columns)

RECOMMENDATION #12: AUTOMATIC SCHEDULING MODULE
--------------------------------------------------------------------------------
HOW TO ENABLE:
  1. Login as Administrator
  2. Go to Settings > System Settings
  3. Find "Automatic Scheduling" toggle
  4. Turn ON

HOW IT WORKS:
  1. When submission is APPROVED
  2. System automatically creates schedule entry
  3. Finds optimal time slot:
     - Checks staff availability
     - Detects conflicts
     - Balances workload
  4. Notifications sent to staff and requestor
  5. Appears on calendar immediately

STAFF AVAILABILITY:
  SET AS STAFF:
    1. Login as staff member
    2. Go to "My Schedule" > "Set Availability"
    3. Click dates on calendar
    4. Choose: Available, Partially Available, Unavailable
    5. Save

  MANAGE AS ADMIN:
    1. Go to "Schedule" > "Staff Availability"
    2. View all staff calendars
    3. Set holidays, training days, vacations
    4. System respects these when auto-scheduling

CONFLICT DETECTION:
  - Prevents double-booking
  - Enforces buffer time (15 min between appointments)
  - Checks staff availability windows
  - Suggests alternatives if conflict

API ENDPOINTS:
  POST /schedule/check-availability (date, time, staff_id)
  GET /schedule/available-slots (date, staff_id)
  GET /schedule/staff-workload (date_from, date_to)

FILES: app/Models/ScheduleModel.php, app/Controllers/Schedule.php

RECOMMENDATION #15 & #16: CALENDAR FILTERS
--------------------------------------------------------------------------------
HOW TO ACCESS CALENDAR:
  1. Navigate to "Schedule" > "Calendar"
  2. Filter toolbar appears at top
  3. Five filter dropdowns available

FILTER OPTIONS:
  1. PRIORITY: All, High, Medium, Low, Not Set
  2. SERVICE: All Services, Equipment Request, Leave Application, etc.
  3. STATUS: All, Submitted, Approved, Pending Service, Completed, Rejected
  4. REQUESTING OFFICE: All Offices, HR, Finance, IT, Operations, etc.
  5. ASSIGNED STAFF: All Staff, John Doe, Jane Smith, etc.

HOW TO FILTER:
  1. Click any dropdown (e.g., "Requesting Office")
  2. Select department
  3. Calendar updates to show only that office's appointments
  4. Combine multiple filters:
     - Example: Priority=High + Office=HR + Status=Pending
     - Shows only high-priority pending HR requests
  5. Click "Clear Filters" to reset

CALENDAR VIEWS:
  - Month View: See entire month
  - Week View: Hour-by-hour schedule
  - Day View: Single day details
  - List View: Table format

EVENT DISPLAY:
  - Color-coded by status and priority
  - Shows: Service name, requestor, time, department
  - Click event for details
  - Drag-and-drop to reschedule

USE CASES:
  - View only your department's appointments
  - Filter high-priority items for urgent attention
  - See specific staff member's schedule
  - Focus on pending service tasks for today

FILES: app/Views/schedule/calendar.php (lines 22-68)
